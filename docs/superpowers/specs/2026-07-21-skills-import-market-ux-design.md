# Skills 导入预览与市场状态设计

## 目标

在不改变现有 Skills 目录结构和 Hermes CLI 安装方式的前提下，补齐两个高频操作的反馈：ZIP 导入执行前明确显示新增、覆盖和跳过项；技能市场明确显示本地已安装状态，并在安装过程中提供逐项进度、成功和错误反馈。

## ZIP 导入冲突预览

ZIP 导入保持现有安全限制，包括压缩包体积、解压后体积、文件数量、路径穿越、符号链接和 Skill 名称校验。后端在现有 `POST /api/skills/import-zip` 上增加 `preview` 查询参数，预览和正式导入共用同一套压缩包扫描及目标路径规划逻辑，避免两套判断产生差异。

预览不创建目录、不写文件、不备份文件，也不清理缓存。返回每个 Skill 的计划动作：

- `add`：目标目录不存在，正式导入会新增。
- `overwrite`：目标目录存在且开启覆盖，正式导入会先备份再覆盖。
- `skip`：目标目录存在且未开启覆盖，正式导入会跳过。

前端采用两阶段流程。选择压缩包及覆盖选项后，用户先点击“预览导入”；预览成功后显示分类计数和逐项动作，才允许点击“确认导入”。更换文件或覆盖选项会立即使旧预览失效，防止用户确认过期结果。正式导入仍会重新校验压缩包和当前文件系统状态，因此实际结果是最终事实。

## 技能市场安装状态

市场搜索结果由后端附加本地安装信息。匹配以市场项的规范化 Skill 名称为准，与 Hermes 当前按名称启用/禁用 Skill 的行为保持一致。返回 `installed`、`installed_category` 和 `installed_path`，不依赖前端猜测目录。

前端在每个市场项上显示状态：

- 未安装：显示“安装”。
- 已安装且未开启强制重装：显示“已安装”，按钮禁用。
- 已安装且开启强制重装：显示“重新安装”，按钮可用。
- 请求进行中：当前项显示“安装中”，其他安装按钮临时禁用。
- 请求成功：当前项立即切换为“已安装”，显示成功反馈，并刷新主 Skills 列表。
- 请求失败：错误显示在对应市场项内，按钮恢复为“重试”，不覆盖其他项目状态。

搜索错误保留在搜索区域，安装错误保留在项目行内，两类错误互不覆盖。

## 数据边界

后端负责压缩包安全校验、冲突动作计算、本地安装状态识别和 Hermes CLI 调用。前端只负责两阶段交互和状态呈现。译文缓存、Skill 删除备份和 Hermes Skills 目录不受本次改动影响。

## 测试与验收

后端测试覆盖预览无写入、三种冲突动作、正式覆盖结果、市场已安装标记和未安装结果。前端静态回归测试覆盖预览请求、确认导入门控、状态文案、已安装禁用、强制重装和逐项反馈。最后运行 Skills 测试、TypeScript/Vite 构建，并在 `3002` 的真实页面验证两个工作流。
